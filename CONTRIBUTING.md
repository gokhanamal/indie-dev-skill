# Contributing

Thanks for contributing to this skill.

## Scope
- Keep changes focused on onboarding and paywall strategy for subscription apps.
- Avoid architecture/tooling/compliance guidance in skill content.

## Skill Requirements
- Each skill folder must contain `SKILL.md`.
- Frontmatter `name` must match the skill directory name.
- Keep reference docs concise, factual, and non-prescriptive.

## Validation
Run local validation before opening a PR:

```bash
.github/scripts/check-skill.sh .
```

## Pull Requests
- Include a clear summary of what changed and why.
- Keep examples practical and avoid guarantees in benchmark claims.
